# DJ Kimchi — Claude Project Instructions

## Project Identity

**What it is:** Official website for DJ Kimchi — Nairobi-based DJ specialising in Afrobeats, Amapiano, Gengetone, and Dancehall.
**Framework:** Next.js 16 App Router — React 19, TypeScript, Tailwind v4, Shadcn/ui
**Package manager:** Bun (use `bun install`, `bun run dev`, `bun run build`)
**Deployed:** Vercel. Set `NEXT_PUBLIC_SITE_URL` in Vercel env vars to the production domain.
**Repo:** https://github.com/rauell1/dj-kimchi.git

---

## Architecture

```
src/app/
  layout.tsx        — root layout with metadata, JSON-LD, Google/Bing verification
  page.tsx          — server component (SSR, NOT "use client")
  sitemap.ts        — /sitemap.xml
  robots.ts         — /robots.txt
  api/
    booking/        — booking form email handler (Resend)
    mixcloud-url/   — proxy to Mixcloud API
    webhooks/inbound/

src/components/site/
  navigation.tsx    — sticky nav, hash-anchor links, theme toggle
  hero.tsx          — hero section
  about.tsx         — about section
  music.tsx         — mixes / HearThis player
  videos.tsx        — YouTube embeds
  photos.tsx        — photo gallery
  bookings.tsx      — booking form (react-hook-form + Resend)
  footer.tsx
  global-player.tsx — persistent audio player

src/lib/
  site-data.ts      — ALL content constants: NAV_LINKS, MIXES, HEARTHIS_TRACKS,
                       VIDEOS, SOCIAL_LINKS, STATS, PHOTOS — edit content here
```

---

## Critical Rules

### Page must stay a Server Component
`src/app/page.tsx` must NOT have `"use client"` at the top. Individual section components handle their own interactivity. Removing SSR breaks Google/Bing indexing.

### Content lives in site-data.ts
All text, links, mixes, photos, and social URLs are constants in `src/lib/site-data.ts`. Never hardcode content in components.

### Navigation uses hash anchors
The site is single-page. Nav links like `#hero`, `#about`, `#music`, `#videos`, `#bookings` are scroll anchors — not routes. Do not add Next.js `<Link>` routing.

### Email
Booking form emails via Resend. Requires `RESEND_API_KEY` env var.

## SEO

- Sitemap: `/sitemap.xml` — generated by `src/app/sitemap.ts`
- Robots: `/robots.txt` — generated by `src/app/robots.ts`
- JSON-LD: `Person` + `MusicGroup` + `WebSite` schema in `layout.tsx`
- Canonical + OG metadata in `layout.tsx`
- Google verification: meta tag `google-site-verification` in `layout.tsx`
- Bing verification: meta tag `msvalidate.01` in `layout.tsx`
- Verification files: `public/BingSiteAuth.xml`, `public/googlee69e3a7319b06c7f.html`
